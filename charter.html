Project Charter

Neovim is not a rewrite of Vim; it is a refactor (and sometimes redactor) in the tradition of Vim which itself derives from Stevie.

Neovim is a continuation and extension of Vim. Many rewrites, clones, emulators and imitators exist; some are very clever; none are Vim. Neovim strives to be a superset of Vim, notwithstanding some intentionally removed misfeatures; thus insofar as one accepts those few and carefully-considered excisions, Neovim is Vim. The project is built for users who want the good parts of Vim, without compromise, plus some new good parts.

The design of Neovim is informed by established industry principles such as modularity, isolation, well-defined interfaces, in order to:

  • Simplify maintenance and comprehension
  • Encourage contributions, scale development work
  • Enable new applications (remote plugins, integrations) and frontends (UIs)

This purpose of this charter is to clarify priorities and to guide decisions.

Project Goals

  • Leverage existing and ongoing Vim development development
  • Maintain feature parity with Vim and avoid regressions
  • Continue the Vim tradition of backwards compatibility
  • Keep the core small and fast
  • Make the core more extensible
  • Target all platforms supported by libuv
  • Optimize "out of the box", for new users but especially regular users
  • Delegate functionality to plugins, but preserve the utility of the nvim core
  • Ship Luajit as first-class scripting alternative

Project Non-Goals

  • Turn Vim into an IDE
  • Limit applications (such as IDEs) built with Neovim
  • Deprecate VimL

Guiding Principles

  • Do Not Regress From Origin
  • Decide outcomes by weighing cost and benefit
  • Use automation to solve problems
  • Enable new contributors: remove barriers to entry
  • Prefer usability over tradition if the benefit greatly outweighs the cost
  • In matters of taste or ambiguity, favor tradition/compatibility
  • Unblock contributors and plugin authors; allow progress
  • Give usability a chance
